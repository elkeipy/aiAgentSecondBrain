# 워크플로우 오케스트레이션 및 핵심 원칙 (Workflow Orchestration & Core Principles)

## 1. 워크플로우 오케스트레이션 (Workflow Orchestration)

### 1. 계획 모드 기본 적용 (Plan Mode Default)
* **적용 대상:** 3단계 이상의 작업이나 아키텍처 결정이 필요한 모든 비사소한(non-trivial) 작업.
* **대응:** 작업 도중 문제가 발생하면 즉시 멈추고 계획을 재수립한다.
* **검증:** 단순히 결과물을 만드는 것에 그치지 않고, 검증 단계 자체를 계획 모드에 포함한다.
* **명확성:** 모호함을 줄이기 위해 시작 전 상세한 사양(spec)을 작성한다.

### 2. 하위 에이전트 전략 (Subagent Strategy)
* **컨텍스트 관리:** 메인 컨텍스트 윈도우를 깔끔하게 유지하기 위해 하위 에이전트를 자유롭게 사용한다.
* **분업:** 리서치, 탐색, 병렬 분석 작업은 하위 에이전트에게 위임한다.
* **집중 실행:** 복잡한 문제일수록 하위 에이전트를 통해 컴퓨팅 자원을 집중 투입하며, 에이전트당 하나의 작업만 전담시켜 실행력을 높인다.

### 3. 자기 개선 루프 (Self-Improvement Loop)
* **피드백 반영:** 사용자의 수정 지시가 있을 때마다 `tasks/lessons.md`를 업데이트하여 패턴화한다.
* **규칙화:** 같은 실수를 반복하지 않도록 스스로를 위한 규칙을 작성한다.
* **지속적 개선:** 실수가 줄어들 때까지 이 과정을 철저하게 반복하며, 세션 시작 시 해당 프로젝트의 교훈(lessons)을 복습한다.

### 4. 완료 전 검증 (Verification Before Done)
* **증명 우선:** 동작이 확인되지 않은 상태에서 작업을 완료로 표시하지 않는다.
* **차이 분석:** 필요한 경우 변경 전후의 동작 차이(Diff)를 분석한다.
* **기준 수립:** "스태프 엔지니어가 이 코드를 승인할 것인가?"를 스스로에게 자문한다.
* **실증:** 테스트를 수행하고 로그를 확인하며 정확성을 증명한다.

### 5. 우아한 해결책 지향 (Demand Elegance)
* **질문:** 비사소한 변경 시, 잠시 멈추고 "더 우아한 방법은 없는가?"를 고민한다.
* **개선:** 해결책이 조잡(hacky)하다고 느껴지면, "현재의 모든 정보를 바탕으로 더 우아한 방식으로 구현하라"고 지시한다.
* **균형:** 간단하고 명백한 수정에는 이 과정을 생략하여 과도한 엔지니어링을 방지한다.
* **자기 검열:** 결과를 제시하기 전에 자신의 작업물을 직접 비판적으로 검토한다.

### 6. 자율적인 버그 수정 (Autonomous Bug Fixing)
* **능동적 해결:** 버그 리포트를 받으면 추가 질문 없이 즉시 해결한다.
* **근거 기반:** 로그, 에러, 실패한 테스트를 분석하여 해결한다.
* **사용자 배려:** 사용자가 문맥을 전환할 필요가 없도록 하며, 지시 없이도 실패하는 CI 테스트를 스스로 수정한다.

## 2. 작업 관리 (Task Management)
* **계획 우선:** `tasks/todo.md`에 체크 가능한 항목으로 계획을 작성한다.
* **계획 검증:** 구현 시작 전 사용자와 계획을 확인한다.
* **진행 추적:** 작업이 진행됨에 따라 항목을 완료 처리한다.
* **변경 설명:** 각 단계마다 고수준의 요약을 제공한다.
* **결과 문서화:** `tasks/todo.md`에 검토(Review) 섹션을 추가한다.
* **교훈 습득:** 수정 후 `tasks/lessons.md`를 업데이트한다.

## 3. 핵심 원칙 (Core Principles)
* **단순성 우선 (Simplicity First):** 모든 변경은 가능한 한 단순하게 만들고, 코드 변경 영향을 최소화한다.
* **근본 해결 (No Laziness):** 임시방편(임시 패치)을 지양하고, 시니어 개발자 수준의 표준을 지키며 근본 원인을 찾는다.
* **최소 영향 (Minimal Impact):** 꼭 필요한 부분만 수정하여 예기치 못한 버그 발생을 방지한다.
